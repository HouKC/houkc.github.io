## 0x00 JavaScript

JavaScript这个部分是要求在输入框提交字符串“success”，返回“Well done!”即为成功。

观察源码可以看到中间一大串代码是MD5加密算法的实现，直接跳过即可。然后是rot13加密函数，也先不用关注。

最核心的是在最后面的function generate_token()处，这个函数会根据变量phrase，去做rot13加密和MD5哈希，生成的值作为token值。而phrase变量是从前端中获取的，也就是用户输入的字符串。

这样一来我们就只需要在输入success给前端，然后调用一下这个generate_token()函数，去生成token，然后再提交即可。

#### 解题思路
我们从上面源码可以看到这段源码是直接放在前端html中，作为script标签存在的。也就是我们只要设置Low级别，转到JavaScript攻击这个页面，就可以调用generate_token函数了。

按照以下步骤操作：
- 首先输入字符串“success”到输入框中
- 接着按F12进入开发者工具
- 打开控制台（Firefox、IE为控制台，Chrome为console）
- 输入generate_token()并回车执行
- 这时再在页面中点击Submit提交，就成功提交了字符串携带token。
返回“Well done!”，提交成功！



## 0x02 Medium

#### 源码分析
在开始源码分析之前，需要注意一件事：
- 如果dvwa网站是直接将源码全部放在网站根目录下，也就是vulnerabilities等文件夹是在www目录下，没事了，可以继续往下看源码了哈哈哈！
- 如果是像我一样，先在网站根目录下创建了一个目录（如dvwa文件夹），然后再将网站源码放在dvwa下，很不幸，你需要修改一下源码才能做这个级别的练习，将/dvwa/vulnerabilities/javascript/source/medium.php中的链接修改成下面这样子：
```php
# 修改前
<?php
$page[ 'body' ] .= <<<EOF
<script src="/vulnerabilities/javascript/source/medium.js"></script>
EOF;
?>
# 修改后，在src处/vulnerabilities前加入自己网站根目录到dvwa源码的路径
<?php
$page[ 'body' ] .= <<<EOF
<script src="/dvwa/vulnerabilities/javascript/source/medium.js"></script>
EOF;
?>
```

```JavaScript
function do_something(e){
	for(var t="",n=e.length-1;n>=0;n--)
		t+=e[n];
		return t
		}
		
setTimeout(function(){
	do_elsesomething("XX")
	},300);

function do_elsesomething(e){
	document.getElementById("token").value=do_something(e+document.getElementById("phrase").value+"XX")
	}
```
与Low级别不同的是，Low级别计算token的js代码是在前端HTML中的，但Medium级别的计算token代码是在js文件里，也就是需要先加载到这个js文件，才能调用相关函数。

Medium级别在HTML前端代码script标签中导入了js文件，而js文件中核心代码是do_elsesomething(e)函数，参照setTimeout()函数中调用的情况，可知需要往do_elsesomething中传参e="XX"。

#### 解题思路
我们想要调用do_elsesomething()函数还需要先把medium.js加载到控制台。加载的方式就是直接在页面中点击一次Submit提交就可以了，这样前端就会去加载相关的js文件来做运算。

所以，
- 首先F12打开开发者模式
- 然后点击页面中的Submit提交按钮提交一次
- 接着在输入框中输入字符串“success”
- 再在控制台输入do_elsesomething("XX")并回车执行
- 这时再在页面中点击Submit提交，就成功提交了字符串携带token。
看到返回”Well done!“，提交成功！
